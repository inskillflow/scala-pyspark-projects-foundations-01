# TP1 — Analyse d'une action boursière avec Spark SQL
### BigData2 — Niveau : Débutant

---

## Contexte

Vous disposez d'un fichier CSV contenant l'historique des cours boursiers de l'action **Apple (AAPL)**.
Le fichier se trouve dans le dossier `data/AAPL.csv`.

Chaque ligne représente une journée de cotation :

```
Date,Open,High,Low,Close,Volume,Adj Close
2015-08-07,114.580002,116.25,114.50,115.519997,38421400,115.519997
2015-08-06,115.970001,116.50,114.120003,115.129997,52903000,115.129997
...
```

| Colonne | Description |
|---|---|
| `Date` | Date de cotation (`yyyy-MM-dd`) |
| `Open` | Prix d'ouverture |
| `High` | Prix le plus haut de la journée |
| `Low` | Prix le plus bas de la journée |
| `Close` | Prix de clôture |
| `Volume` | Nombre d'actions échangées |
| `Adj Close` | Prix de clôture ajusté (dividendes, splits) |

---

## Objectif

Écrire un programme Scala/Spark qui charge ce fichier et répond aux 6 requêtes ci-dessous en utilisant **Spark SQL** (`spark.sql(...)`).

> **Contrainte importante :** La colonne `Date` doit rester de type `String` — ne pas la convertir en `DateType`.
> L'année et le mois seront extraits avec la fonction SQL `SUBSTRING`.

---

## Travail demandé

### Étape 1 — Mise en place (3 pts)

1. Créer une `case class Stock` avec les 7 champs du CSV
2. Écrire une fonction `parseStock(line: String): Stock` qui parse une ligne CSV
3. Écrire une fonction `parseRDD(rdd: RDD[String]): RDD[Stock]` qui filtre l'en-tête et mappe les lignes
4. Charger le fichier, créer un DataFrame, renommer la colonne `Date` (si nécessaire) et l'enregistrer comme vue temporaire `stocks`
5. Afficher le schéma et les 20 premières lignes

### Étape 2 — Requêtes Spark SQL (12 pts)

Écrire et exécuter les requêtes SQL suivantes. Pour chaque requête, afficher le résultat avec `.show()`.

**Requête 1 (2 pts)** — Liste des dates, prix d'ouverture et prix de clôture
```
Colonnes attendues : Date | Open | Close
```

**Requête 2 (2 pts)** — Pour chaque journée, calculer la différence entre le prix de clôture et le prix d'ouverture (arrondie à 2 décimales), appelée `variation`
```
Colonnes attendues : Date | variation
```

**Requête 3 (2 pts)** — Volume maximum et volume minimum sur toute la période
```
Colonnes attendues : max_volume | min_volume
```

**Requête 4 (2 pts)** — Moyenne du prix d'ouverture par année, triée par année croissante
```
Colonnes attendues : annee | moyenne_ouverture
Indice : utiliser SUBSTRING(Date, 1, 4)
```

**Requête 5 (2 pts)** — Somme des volumes échangés par mois (tous années confondues), triée par mois
```
Colonnes attendues : mois | total_volume
Indice : utiliser SUBSTRING(Date, 6, 2)
```

**Requête 6 (2 pts)** — Les 5 journées avec le prix de clôture le plus élevé
```
Colonnes attendues : Date | Close
```

### Étape 3 — Question bonus (5 pts)

**Requête 7** — Pour chaque année, afficher le prix de clôture maximum, minimum et moyen, triés par année.
```
Colonnes attendues : annee | max_close | min_close | avg_close
```
