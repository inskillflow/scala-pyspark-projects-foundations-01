# TP2 — Analyse et comparaison de plusieurs actions boursières
### BigData2 — Niveau : Intermédiaire

---

## Contexte

Vous disposez de **5 fichiers CSV** correspondant à 5 grandes entreprises technologiques américaines :

| Fichier | Entreprise | Symbole |
|---|---|---|
| `data/AAPL.csv` | Apple | AAPL |
| `data/GOOG.csv` | Google (Alphabet) | GOOG |
| `data/MSFT.csv` | Microsoft | MSFT |
| `data/AMZN.csv` | Amazon | AMZN |
| `data/NFLX.csv` | Netflix | NFLX |

Chaque fichier a le même format :
```
Date,Open,High,Low,Close,Volume,Adj Close
2015-08-07,114.580002,116.25,114.50,115.519997,38421400,115.519997
```

---

## Objectif

Créer un programme Scala/Spark qui **combine tous les fichiers** dans un seul DataFrame en ajoutant une colonne `symbol` (le nom de l'action), puis répond aux requêtes SQL demandées.

---

## Travail demandé

### Étape 1 — Chargement multi-fichiers (5 pts)

1. Utiliser `spark.read.csv(...)` avec `header = true` et `inferSchema = true` pour charger chaque fichier
2. Ajouter une colonne `symbol` à chaque DataFrame avec `withColumn("symbol", lit("AAPL"))` etc.
3. **Fusionner** les 5 DataFrames en un seul avec `.union()` ou `.unionAll()`
4. Enregistrer le DataFrame fusionné comme vue temporaire `stocks_all`
5. Afficher le schéma et le nombre total de lignes

> **Indice :** La colonne s'appelle `Adj Close` (avec un espace) — dans Spark SQL, l'entourer de backticks :
> `` `Adj Close` ``

### Étape 2 — Requêtes de comparaison (12 pts)

**Requête 1 (2 pts)** — Nombre de jours de cotation par action
```
Colonnes attendues : symbol | nb_jours
Triée par : nb_jours DESC
```

**Requête 2 (2 pts)** — Prix de clôture maximum atteint par chaque action sur toute leur histoire
```
Colonnes attendues : symbol | max_close
Triée par : max_close DESC
```

**Requête 3 (3 pts)** — Moyenne annuelle du volume échangé par action et par année, pour les années 2010 à 2015
```
Colonnes attendues : symbol | annee | avg_volume
Triée par : symbol, annee
```

**Requête 4 (3 pts)** — Pour chaque action, la journée où elle a eu le **plus grand écart** entre `High` et `Low` (volatilité intra-journalière maximale)
```
Colonnes attendues : symbol | Date | ecart_max
```

**Requête 5 (2 pts)** — Compter combien de jours chaque action a eu une variation positive (Close > Open) et combien de jours une variation négative
```
Colonnes attendues : symbol | jours_hausse | jours_baisse
```

### Étape 3 — Jointure et comparaison (3 pts)

**Requête 6** — Trouver tous les jours où **AAPL et MSFT ont toutes les deux** une variation positive le même jour (Close > Open pour les deux).
Afficher la date et la variation des deux actions.
```
Colonnes attendues : Date | variation_AAPL | variation_MSFT
Triée par : Date DESC
Limitée à : 20 résultats
```

> **Indice :** Utiliser deux sous-requêtes ou deux vues temporaires, puis faire un JOIN sur la date.

## Rappels techniques

```scala
// Charger un CSV avec inferSchema
val df = spark.read
  .option("header", "true")
  .option("inferSchema", "true")
  .csv("data/AAPL.csv")

// Ajouter une colonne constante
import org.apache.spark.sql.functions.lit
val dfAvecSymbol = df.withColumn("symbol", lit("AAPL"))

// Fusionner deux DataFrames (même schéma)
val dfFusion = df1.union(df2).union(df3)

// Colonne avec espace dans Spark SQL : utiliser des backticks
spark.sql("SELECT `Adj Close` FROM stocks_all")

// Compter les lignes
dfFusion.count()
```
